# 技能-前台頁面管理

## 目的

建立前台頁面可由後台管理的能力。

## 第一版範圍

1. 頁面新增 / 編輯 / 刪除
2. 拖曳排序
3. 父子層設定
4. 顯示開關
5. 前台導覽自動渲染

## 資料模型建議

### `site_pages`

- `title`
- `slug`
- `parent_id`
- `sort_order`
- `is_visible`
- `page_type`
- `template_key`
- `nav_label`
- `nav_icon`
- `seo_title`
- `seo_description`

## 限制

1. 最多 2 層
2. 不做任意深度樹
3. 不先做 block-level page builder
4. 先做頁面結構管理，不先做完整 CMS

## `site_pages` 第一版 schema

第一版先以「可管理導覽結構」為主，不等 Notion / n8n 完成。

### 核心欄位

| 欄位 | 型別 | 必填 | 用途 |
| --- | --- | --- | --- |
| `title` | string | 是 | 後台管理用頁面名稱 |
| `slug` | string | 是 | 前台 section key / 路由鍵，第一版需唯一 |
| `parent_id` | string \| null | 否 | 父頁 ID，最多只支援 2 層 |
| `sort_order` | number | 是 | 同層排序 |
| `is_visible` | boolean | 是 | 是否顯示於前台導覽 |
| `page_type` | string | 是 | `section` / `group` / `external` |
| `template_key` | string | 是 | 對應模板或 section 類型 |
| `nav_label` | string | 是 | 導覽顯示文字 |
| `nav_icon` | string | 否 | 導覽 icon / emoji |
| `seo_title` | string | 否 | SEO 標題 |
| `seo_description` | string | 否 | SEO 描述 |

### 實作備註

1. 第一版可額外保留 `id` 作為本地 mock 與後續 Notion 主鍵映射使用
2. `group` 類型只負責導覽分組，不要求對應實際內容頁
3. `external` 類型第一版可使用 `slug` 存相對或絕對 URL，例如 `leyi/achievements.html`
4. 若父頁被隱藏，子頁在第一版也應一併隱藏
5. 子頁的父層只能是頂層頁，避免超過 2 層

## 第一版實作策略

1. 先以本地設定或 mock data 跑通後台管理與前台導覽
2. 後台先提供列表、排序、父層、顯示開關、基本編輯入口
3. 前台先改成由 `site_pages` 生成導覽，不先做完整內容動態渲染
4. 後續再把資料來源替換到 Notion / n8n

## 排序與刪除規則

1. 第一版拖曳排序只支援同一父層內排序
2. 跨父層搬移先透過編輯表單調整 `parent_id`
3. 刪除父頁時，本地 mock 會一併移除其子頁
4. 正式串接資料庫前，所有刪除與排序都只作用在本地 `localStorage`
